0. React とは — なぜ「コンポーネント」なのか

React は「ユーザーインターフェース（画面）」を作るための JavaScript ライブラリです。 React の世界では、画面をコンポーネント（component＝部品）という小さな単位に分けて組み立てます。

たとえば普通の Web ページを思い浮かべてください。ヘッダー、検索ボックス、商品カード、フッター……。 これらを1つずつ「部品」として作り、それを組み合わせてページ全体にします。

なぜ部品にするのか:
  • 再利用できる: 「商品カード」を1個作れば、100個でも map で量産できる。
  • 分かりやすい: 巨大な HTML を1枚で書くより、小さな部品に分けた方が読みやすく直しやすい。
  • データと見た目をまとめられる: 「このデータをこう表示する」を部品の中に閉じ込められる。

そして React のコンポーネントは、特別なものではなくただの JavaScript の関数です。 「画面の見た目（JSX）を return する関数」、これが React コンポーネントの正体です。

1. 準備: ビルドなしで React を動かす仕組み（CDN + Babel）

本格的な React 開発では npm で道具一式をインストールしますが、今回は学習に集中するため、インストール不要の最短ルートで動かします。 HTML の中で次の3つを読み込むだけです。

<!-- ① React 本体（コンポーネントを作る機能） -->
<script src="https://unpkg.com/react@18/umd/react.development.js"></script>
<!-- ② ReactDOM（作った部品を実際の画面に描く機能） -->
<script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
<!-- ③ Babel（JSX をブラウザが分かる JS に翻訳する道具） -->
<script src="https://unpkg.com/@babel/standalone@7/babel.min.js"></script>

そして自分のコードは <script type="text/babel"> の中に書きます。 こうすると Babel がその場で JSX を翻訳してくれます。

用語: Babel（バベル）とは — 「新しい書き方や JSX を、ブラウザが理解できる普通の JavaScript に翻訳する道具」です。 ブラウザは JSX（後述）をそのままでは読めないので、翻訳役が必要になります。今回は手軽さ優先でブラウザ上で翻訳していますが、 実務ではあらかじめ翻訳（ビルド）しておく方式（Vite など）を使います。これは Day12 以降で扱います。
注意: この CDN 方式は学習用です。動作はネット接続が必要で、本番では使いません。 「まず React の考え方に慣れる」ことが目的です。

2. 最初のコンポーネント — 「JSX を返す関数」

コンポーネントは「JSX を return する関数」です。さっそく一番小さい例を見ましょう。

function Greeting() {
  return <h1>こんにちは、React!</h1>;
}

この <h1>こんにちは、React!</h1> の部分が JSX です。 HTML のように見えますが、これは JavaScript の中に書ける「見た目を表す式」です（詳しくは セクション4）。

超重要ルール: コンポーネント名は必ず大文字で始める。 Greeting は OK、greeting は NG です。 React は「大文字で始まる名前＝自作コンポーネント」「小文字＝普通の HTML タグ（div など）」として区別します。 小文字で書くと、ただの HTML タグだと誤解されて表示されません。

複数行の JSX を返すときは、丸カッコ ( ) で囲むのが定番です（return の直後で改行するとエラーになりやすいため）。

function Profile() {
  return (
    <div>
      <h2>山田 太郎</h2>
      <p>React を学習中です</p>
    </div>
  );
}

3. 画面に表示する — createRoot と render

コンポーネントを作っただけでは画面には出ません。「どの場所に」「何を」描くかを ReactDOM に指示します。

<!-- HTML 側: 描画先となる「空の箱」を用意 -->
<div id="root"></div>
// JS 側: その箱に Greeting を描く
const root = ReactDOM.createRoot(document.getElementById("root"));
root.render(<Greeting />);
1行ずつ:
  • document.getElementById("root") … HTML 内の id="root" の箱を取得。
  • ReactDOM.createRoot(...) … その箱を「React が管理する描画先（root）」にする。
  • root.render(<Greeting />) … その中に Greeting コンポーネントを描く。
自作コンポーネントは <Greeting /> のように「タグの形」で呼び出すのがポイントです（Greeting() と関数呼び出しはしません）。

4. JSX のルール（HTML との違い）

JSX は「JavaScript の中に HTML のような見た目を書ける記法」です。便利ですが、HTML と少しだけ違うルールがあります。 （JSX 自体は Day05 でさらに深掘りします。ここでは最低限を押さえます。）

ルール説明例
① 必ず1つの要素で包む return する JSX は「親1個」にまとめる。並べたいときは <div> か空タグ <></> で囲む。 <>...</>
② タグは必ず閉じる HTML で省略できる <br> なども、JSX では <br /> と閉じる。 <img src="..." />
③ class ではなく className class は JS の予約語なので、CSS クラスは className と書く。 <p className="msg">
空タグ <></> とは — 「フラグメント」と呼ばれる、画面に余計な箱を作らずに複数要素をまとめるための入れ物です。 「1個に包みたいけど div は増やしたくない」ときに使います。

5. 波括弧 { } で JavaScript を埋め込む

JSX の中で { } を書くと、その中は JavaScript として評価されます。 変数の値や計算結果を、見た目の中に差し込めます。

function Welcome() {
  const name = "Taro";
  const year = 2026;
  return <p>{name} さん、{year} 年へようこそ！</p>;
  //        ↑ 変数      ↑ 変数
}
// 表示: 「Taro さん、2026 年へようこそ！」

{ } の中には「値になるもの（式）」なら何でも書けます。計算、関数呼び出し、三項演算子などもOKです。

function Price() {
  const price = 1200;
  return <p>税込: {Math.floor(price * 1.1)} 円</p>; // 計算もできる → 税込: 1320 円
}

条件で表示を変える（三項演算子）

「点数が60以上なら合格、そうでなければ不合格」のように、条件によって表示を切り替えたい場面はとても多いです。 JavaScript には if 文がありますが、先ほど触れたとおり JSX の { } の中には「文」は入れられず「式（値になるもの）」しか書けません。 そこで活躍するのが 三項演算子（さんこうえんざんし / ternary operator）です。これは「if と else を1つの式にまとめたもの」と考えてください。

書き方

条件 ? 条件がtrueのときの値 : 条件がfalseのときの値

「?（もし〜なら）」と「:（そうでなければ）」の2つの記号で3つの部分をつなぐので「三項」演算子と呼びます。 読むときは「条件 ？ こっち ： あっち」と覚えると分かりやすいです。

if 文と見比べる

// ① if 文（これは「文」。値にならないので JSX の { } には入れられない）
let label;
if (score >= 60) {
  label = "合格";
} else {
  label = "不合格";
}

// ② 三項演算子（これは「式」。1つの値になるので { } に入れられる）
const label = score >= 60 ? "合格" : "不合格";
同じことを表しています。 ① の「if で label に代入する」処理を、② は1行の式に凝縮しています。 score が 72 なら "合格"、49 なら "不合格" という1つの値に決まるので、そのまま { } に入れられます。

JSX の中で使う

function Result() {
  const score = 72;
  return <p>{score >= 60 ? "合格" : "不合格"}</p>;
  //          └─ 条件 ─┘   └true┘   └false┘
}
// score が 72 なので「合格」と表示される

返す値は文字列だけでなく、JSX そのものでも構いません（タグを丸ごと切り替えられます）:

function Status() {
  const isLoggedIn = true;
  return (
    <div>
      {isLoggedIn
        ? <span>ようこそ！</span>   // true のとき返す JSX
        : <a href="/login">ログイン</a>} // false のとき返す JSX
    </div>
  );
}

map と組み合わせる（演習5で使います）

三項演算子は map の中で「1件ごとに表示を変える」のにそのまま使えます:

const students = [
  { id: 1, name: "Taro",   score: 82 },
  { id: 2, name: "Hanako", score: 49 },
];

// 各 student の score を見て「合格 / 不合格」を出し分ける
{students.map((student) => (
  <li key={student.id}>
    {student.name}: {student.score >= 60 ? "合格" : "不合格"}
  </li>
))}
// Taro: 合格 / Hanako: 不合格
「片方だけ表示／非表示」したいとき: 「条件が true のときだけ出す（false なら何も出さない）」場合は、 三項演算子の代わりに && を使うこともできます。例: {isError && <p>エラーです</p>}。 ただし数値の 0 など「falsy な値」が左にあると 0 がそのまま表示される落とし穴があるので、慣れるまでは三項演算子の方が安全です。
{ } に入れられるのは「式（値になるもの）」だけ。 if 文や for 文のような「文」はそのままでは入れられません。 条件分岐は三項演算子、繰り返しはmap（次のセクション）を使うのが React 流です。

6. コンポーネントの入れ子（合成）

コンポーネントの真価は「部品を組み合わせて大きな部品を作れる」ことです。 作ったコンポーネントは、別のコンポーネントの中でタグのように使えます。

function Header() {
  return <header>サイトのヘッダー</header>;
}

function Footer() {
  return <footer>© 2026</footer>;
}

// Header と Footer を「部品」として組み合わせる
function Page() {
  return (
    <div>
      <Header />
      <main>ここが本文です</main>
      <Footer />
    </div>
  );
}
これが「合成（composition）」です。 小さな部品（Header / Footer）を組み合わせて、 大きな部品（Page）を作る。React のアプリは、結局この「部品の入れ子」を積み重ねた1本の木（ツリー）になります。
今回はまだ「外からデータを渡す」方法（props）を学んでいないので、各部品は中に固定のデータを持っています。 「部品に値を渡して中身を変える」のは Day06（Props）で学びます。今日は「作る・組み合わせる・繰り返す」に集中しましょう。

7. リストを map で描く（key の話）

Day03 で学んだ map が、ここでついに React で活躍します。 配列を「JSX 要素の配列」に変換すれば、そのまま画面にリスト表示できます。

function FruitList() {
  const fruits = ["りんご", "みかん", "ぶどう"];
  return (
    <ul>
      {fruits.map((fruit) => (
        <li key={fruit}>{fruit}</li>
      ))}
    </ul>
  );
}
流れ: ["りんご","みかん","ぶどう"] を map で [<li>りんご</li>, <li>みかん</li>, <li>ぶどう</li>] というJSX の配列に変換し、 <ul> の中に { } で埋め込んでいます。Day03 の map がそのまま使えています。
key（キー）を必ず付ける。 map で要素を並べるときは、各要素に key という特別な属性を付けます。 React が「どれがどれか」を見分けて効率よく更新するための目印です。 普通はデータが持つ一意な id（例: key={user.id}）を使います。無いと警告が出ます。

オブジェクトの配列なら、id を key に使うのが定番です:

const users = [
  { id: 1, name: "Taro" },
  { id: 2, name: "Hanako" },
];
// ...
{users.map((user) => (
  <li key={user.id}>{user.name}</li>
))}
