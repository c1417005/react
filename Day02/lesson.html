0. 準備運動: オブジェクトと配列のおさらい

分割代入の前に、「箱」の形を確認します。初学者向けに基礎から行きます。

配列（array）= 順番付きの並び

const fruits = ["りんご", "みかん", "ぶどう"];
// 取り出すときは「番号（インデックス）」を使う。番号は 0 から始まる。
console.log(fruits[0]); // "りんご"
console.log(fruits[1]); // "みかん"
console.log(fruits.length); // 3 （要素の個数。() は付けない＝プロパティ）

オブジェクト（object）= 名前付きの箱

const user = { name: "Taro", age: 20 };
// 取り出すときは「キー（名前）」を使う。
console.log(user.name); // "Taro"
console.log(user.age);  // 20
for...of の補足（よく使うので先に）: 配列を1個ずつ取り出して回すループです。 
const nums = [10, 20, 30];
for (const n of nums) {   // nums から要素を1つずつ n に入れて繰り返す
  console.log(n);         // 10 → 20 → 30
}
for (let i=0; i<nums.length; i++) の「番号で回す形」と結果は同じですが、 中身そのものが欲しいだけなら for...of の方が短く安全です（番号の付け間違いが起きない）。

1. 配列の分割代入

「分割代入（destructuring）」は、配列やオブジェクトの中身をまとめて別々の変数に取り出す書き方です。

今までの書き方

const colors = ["red", "green", "blue"];
const first  = colors[0];
const second = colors[1];
// 1行ずつ番号で取り出すのは面倒…

分割代入の書き方

const colors = ["red", "green", "blue"];
const [first, second, third] = colors;
//     ↑ 左辺を「配列の形」にすると、同じ位置の要素が入る
console.log(first);  // "red"
console.log(second); // "green"
console.log(third);  // "blue"
ポイント: 配列の分割代入は「位置（順番）」で対応します。 左に書いた変数名は自由ですが、入る値は並び順で決まります。

いらない要素はカンマで飛ばせます:

const [ , , third] = colors; // 最初の2つを飛ばす
console.log(third); // "blue"

2. オブジェクトの分割代入

オブジェクトは「キー名」で対応します。順番は関係ありません。

const user = { name: "Taro", age: 20, city: "Tokyo" };

// 今までの書き方
const name = user.name;
const age  = user.age;

// 分割代入（左辺を「オブジェクトの形」にする）
const { name, age } = user;
//      ↑ user の中の name キー・age キーを同じ名前の変数に取り出す
console.log(name); // "Taro"
console.log(age);  // 20
React で超頻出: コンポーネントが受け取る props は、こうやって分割代入で取り出します。 
// 後の Day で出てくる形（今は雰囲気だけ）

3. デフォルト値・名前の付け替え・ネスト

デフォルト値（値が無いときの初期値）

const { name, role = "一般" } = { name: "Taro" };
// role キーが無いので、デフォルトの "一般" が入る
console.log(role); // "一般"

名前の付け替え（リネーム）

const { name: userName } = { name: "Taro" };
// name キーの値を、userName という変数名で受け取る
console.log(userName); // "Taro"
// console.log(name); // ← これはエラー（name という変数は作られていない）

ネスト（入れ子）の取り出し

const data = { user: { name: "Taro", age: 20 } };
const { user: { name } } = data;
console.log(name); // "Taro"

4. スプレッド構文（配列）

スプレッド ... は、配列やオブジェクトの中身を「展開（ばらまく）」する記号です。 分割代入と見た目が似ていますが、役割は逆（取り出す→展開して入れる）です。

配列のコピー

const original = [1, 2, 3];
const copy = [...original]; // original の中身を展開して新しい配列に入れる
console.log(copy); // [1, 2, 3]

// これは「別物の新しい配列」。元には影響しない。
copy.push(4);
console.log(original); // [1, 2, 3] ← 変わらない！

配列の結合・要素の追加

const a = [1, 2];
const b = [3, 4];
const merged = [...a, ...b];      // [1, 2, 3, 4]
const added  = [0, ...a, 99];     // [0, 1, 2, 99]
const copy = original; と書くと、同じ配列を指すだけでコピーになりません。 copy.push() が original も変えてしまいます。だから [...original] で「新しい箱」を作るのが重要です。

5. スプレッド構文（オブジェクト）

オブジェクトのコピー

const user = { name: "Taro", age: 20 };
const copy = { ...user }; // 中身を展開して新しいオブジェクトに
console.log(copy); // { name: "Taro", age: 20 }

一部だけ上書きした新オブジェクトを作る（最重要）

const user = { name: "Taro", age: 20 };
const updated = { ...user, age: 21 };
//                 ↑既存を全部展開  ↑同じキーは「後に書いた方」で上書き
console.log(updated); // { name: "Taro", age: 21 }
console.log(user);    // { name: "Taro", age: 20 } ← 元は無傷
この「展開して一部だけ上書き」が、React の state 更新でそのまま使われます。 state は直接書き換えず、必ず「新しいオブジェクトを作って差し替える」のが React の約束事です（Day10 で正式に学びます）。

6. レスト（残り）構文

同じ ... ですが、左辺（受け取る側）で使うと「残り全部」を集める意味になります。 （スプレッド=ばらまく / レスト=集める。場所で役割が変わります。）

配列の残りを集める

const [head, ...rest] = [1, 2, 3, 4];
console.log(head); // 1
console.log(rest); // [2, 3, 4] ← 残りが新しい配列に集まる

オブジェクトの残りを集める

const { name, ...others } = { name: "Taro", age: 20, city: "Tokyo" };
console.log(name);   // "Taro"
console.log(others); // { age: 20, city: "Tokyo" }

関数の「可変個の引数」を集める

const sum = (...nums) => {
  // nums は受け取った引数を全部集めた配列になる
  let total = 0;
  for (const n of nums) total += n;
  return total;
};
console.log(sum(1, 2, 3));    // 6
console.log(sum(10, 20));     // 30
場所意味例
右辺・呼び出し側スプレッド（展開）[...a], fn(...args)
左辺・引数定義レスト（集める）[x, ...rest], (...args) =>

7. なぜ React で重要なのか（不変性）

React は「state（データ）が前と別物に変わったか」を見て、画面を描き直すか判断します。 このとき、元のオブジェクト/配列を直接書き換える（ミューテーション）と、React が「変わった」と気づけないことがあります。

// ❌ ダメな例（元を直接いじる）
user.age = 21;
list.push(newItem);

// ✅ 良い例（新しい箱を作る = 不変性 immutability）
const newUser = { ...user, age: 21 };
const newList = [...list, newItem];
今日の分割代入・スプレッド・レストは、まさにこの「新しい箱を作って一部だけ変える」ための道具です。 ここを手に馴染ませておくと、Day08 以降の state 更新が一気に楽になります。
