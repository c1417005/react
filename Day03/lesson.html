0. 準備運動: コールバック関数とは

配列メソッドを理解するカギは「コールバック関数」です。 これは「関数の引数として渡す関数」のことです。聞き慣れないと身構えますが、やっていることは単純です。

// 関数は「値」なので、別の関数に渡すことができる。
function greet(name) {
  return "こんにちは " + name;
}

// greet を「呼び出した結果」ではなく、「greet そのもの」を渡す例
function run(fn) {     // fn には関数が入ってくる
  return fn("Taro");   // 受け取った関数を、中で呼び出す
}
console.log(run(greet)); // "こんにちは Taro"
違いに注意: greet（カッコなし）は「関数そのもの」を指します。 greet("Taro")（カッコあり）は「その場で呼び出した結果（文字列）」です。 配列メソッドにはカッコなしの関数（＝後で呼んでもらう関数）を渡します。

実際にはその場で書く「無名のアロー関数」を渡すことがほとんどです:

// 上の run に、その場で作った関数を直接渡す
console.log(run((name) => "やあ " + name)); // "やあ Taro"

1. forEach — 1個ずつ処理する

forEach は、配列の要素を1個ずつ取り出して、渡した関数を実行するだけのメソッドです。 for...of ループの「メソッド版」と考えてください。

const fruits = ["りんご", "みかん", "ぶどう"];

fruits.forEach((fruit) => {
  // fruit に "りんご" → "みかん" → "ぶどう" が順番に入って、関数が3回呼ばれる
  console.log(fruit);
});
// 出力: りんご / みかん / ぶどう

第2引数で「インデックス（番号）」も受け取れます:

fruits.forEach((fruit, index) => {
  console.log(index + ": " + fruit); // "0: りんご" など
});
forEach は戻り値がありません（常に undefined）。「処理を実行するだけ」で、 新しい配列は作りません。新しい配列が欲しいときは次の map を使います。

2. map — 全要素を「変換」して新しい配列を作る（最重要）

map（マップ）は、各要素を変換した結果を集めた「新しい配列」を返すメソッドです。 「全部に同じ加工をして、新しい配列を作りたい」ときに使います。

今までの書き方（for ループ）

const nums = [1, 2, 3];
const doubled = [];
for (const n of nums) {
  doubled.push(n * 2);   // 2倍して新しい配列に詰める
}
console.log(doubled); // [2, 4, 6]

map の書き方

const nums = [1, 2, 3];
const doubled = nums.map((n) => n * 2);
//                        ↑ 各要素 n を「n * 2」に変換した結果が集まる
console.log(doubled); // [2, 4, 6]
console.log(nums);    // [1, 2, 3] ← 元は変わらない（新しい配列を返すだけ）
map のしくみ: 渡した関数が各要素ごとに呼ばれ、そのreturn した値が新しい配列の同じ位置に入ります。 要素数は必ず元と同じ（3個なら3個）になります。return を書き忘れると undefined が並ぶので注意。

オブジェクトの配列を加工する例（React で頻出の形）:

const users = [
  { id: 1, name: "Taro" },
  { id: 2, name: "Hanako" },
];
// 名前だけ取り出した配列にする
const names = users.map((user) => user.name);
console.log(names); // ["Taro", "Hanako"]

3. filter — 条件に合うものだけ残す

filter（フィルター）は、渡した関数が true を返した要素だけを集めた新しい配列を返します。 「ふるい分け」です。要素数は減ることがあるのが map との違いです。

const nums = [1, 2, 3, 4, 5, 6];

// 偶数だけ残す（n % 2 === 0 が true のものだけ通す）
const evens = nums.filter((n) => n % 2 === 0);
console.log(evens); // [2, 4, 6]
%（剰余）の補足: 割った「余り」を出す記号です。4 % 2 は 0（割り切れる＝偶数）、 5 % 2 は 1（余り1＝奇数）。=== 0 で「割り切れるか」を判定しています。

オブジェクトの配列を条件で絞る例:

const users = [
  { name: "Taro",   active: true },
  { name: "Hanako", active: false },
  { name: "Jiro",   active: true },
];
const activeUsers = users.filter((u) => u.active === true);
console.log(activeUsers.length); // 2

4. find / some / every — 探す・判定する

find — 最初に見つかった「1個」を返す

const users = [
  { id: 1, name: "Taro" },
  { id: 2, name: "Hanako" },
];
const found = users.find((u) => u.id === 2);
console.log(found); // { id: 2, name: "Hanako" }
// 見つからなければ undefined を返す
filter は条件に合う全部の「配列」、find は最初の「1個」を返します。 「1件だけ欲しい」なら find の方が分かりやすく、見つかった時点で探索を止めるので無駄がありません。

some / every — true / false で答える

const nums = [1, 2, 3, 4];

// some: 「1つでも」条件を満たすか？
console.log(nums.some((n) => n > 3));  // true（4 がある）

// every: 「全部が」条件を満たすか？
console.log(nums.every((n) => n > 0)); // true（全部 0 より大きい）
console.log(nums.every((n) => n > 2)); // false（1, 2 がある）

5. reduce — 全部を1つの値にまとめる

reduce（リデュース）は、配列全体を「1つの値」に畳み込むメソッドです。 合計・最大値・1つのオブジェクトへの集約など、「最後に1個の答えにしたい」ときに使います。 少し難しいので、ゆっくり見ましょう。

const nums = [1, 2, 3, 4];

const total = nums.reduce((acc, n) => {
  // acc = これまでの累計（accumulator）/ n = 今の要素
  return acc + n;
}, 0); // ← 0 は acc の「初期値」

console.log(total); // 10
動きを追う（初期値 0 のとき）:
回acc（累計）n（今の要素）return（次の acc）
1回目011
2回目123
3回目336
4回目6410
毎回 return した値が「次の回の acc」になります。最後の acc（10）が答えになります。
第2引数の初期値は省略しないのが安全です。 省略すると配列の先頭要素が初期値になり、 空配列のときにエラーになるなど挙動が変わります。慣れるまでは必ず初期値を書きましょう。

6. メソッドチェーン（つなげて使う）

map や filter は「新しい配列」を返すので、その結果にさらにメソッドをつなげて書けます。 これを「メソッドチェーン」と呼びます。

const nums = [1, 2, 3, 4, 5, 6];

const result = nums
  .filter((n) => n % 2 === 0)  // まず偶数だけ → [2, 4, 6]
  .map((n) => n * 10);         // それを10倍   → [20, 40, 60]

console.log(result); // [20, 40, 60]
読み方: 上から順に「データが加工されて流れていく」とイメージします。 filter でふるいにかけ → 残ったものを map で変換、という処理の流れがそのまま読めるのが利点です。 for ループで同じことを書くより、意図が明確になります。

7. 非同期処理とは — なぜ必要か

ここからは話題が変わって「非同期（asynchronous）処理」です。 これは「時間のかかる処理を、待っている間に他のことを止めない」しくみです。

例えば「サーバーからデータを取ってくる」のには時間がかかります。 もしその間プログラム全体が止まったら、画面が固まって何も操作できません。 そこで「結果は後で受け取るね」と予約して、待っている間は先に進めるようにします。

console.log("1: 注文する");

setTimeout(() => {
  // setTimeout は「指定ミリ秒後に関数を実行」する。ここでは1秒後。
  console.log("3: 料理が届く（1秒後）");
}, 1000);

console.log("2: 他のことをする");

// 実際の出力順:
// 1: 注文する
// 2: 他のことをする   ← 待たずに先に進む！
// 3: 料理が届く（1秒後）
ポイント: 上から順に全部待つ（同期）のではなく、時間のかかる処理は脇に置いて先に進み、終わったら結果が返ってくる。 これが非同期です。「3」が最後に出るのが、その証拠です。

8. Promise の基本

Promise（プロミス＝約束）は、「今はまだ無いけど、いずれ結果が出る値」を表す箱です。 結果は2通り: 成功（resolve）か失敗（reject）。

// 「1秒後に成功する」Promise を作る例
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("データ取得成功!"); // 成功した値を渡す
    // reject("エラー発生");    // 失敗ならこちら
  }, 1000);
});

結果を受け取るには .then()（成功時）と .catch()（失敗時）をつなぎます:

promise
  .then((value) => {
    console.log("成功:", value); // "成功: データ取得成功!"
  })
  .catch((error) => {
    console.log("失敗:", error);
  });
多くの「時間のかかる関数」は、最初から Promise を返してくれます（例: ブラウザの fetch()）。 自分で new Promise を書く機会は少なく、返ってきた Promise をどう受け取るかの方が大事です。

9. async / await（読みやすい書き方）

.then() のチェーンは深くなると読みにくくなります。 そこで async / await を使うと、非同期処理を「上から順に読める普通のコード」のように書けます。

// async を付けた関数の中でだけ await が使える
async function main() {
  console.log("取得開始");

  // await: Promise の結果が出る「まで待って」、中身を取り出す
  const value = await promise;   // resolve された値がそのまま value に入る
  console.log("結果:", value);

  console.log("続きの処理");
}
main();
await の意味: 「この Promise が終わるまでここで待ち、成功した値を取り出す」。 .then() の中に書いていた処理を、そのまま下の行に続けて書けるのが利点です。 ただし await は async 関数の中でしか使えません。

エラー処理は try / catch で

async function main() {
  try {
    const value = await promise;  // 失敗するとここで例外が投げられる
    console.log("成功:", value);
  } catch (error) {
    console.log("失敗:", error);   // reject された値がここに来る
  }
}
書き方成功時失敗時
Promise.then().catch()
async/awaitconst x = await ...try { } catch { }

10. なぜ React で重要なのか

  • map: 配列データを画面のリスト（要素の並び）に変換するとき、React では items.map(...) で要素を量産します。これは毎回使います。
  • filter: 「完了済みだけ表示」「検索でヒットしたものだけ表示」など、表示するデータの絞り込みに使います。
  • 不変性との相性: map/filter は元の配列を壊さず新しい配列を返すので、Day02 で学んだ「不変性」とそのまま噛み合います。state 更新で多用します。
  • async/await: サーバーからデータを取得して画面に出す（Effects / データ取得）処理で使います。
// 後の Day で出てくる形（今は雰囲気だけ）
function UserList({ users }) {
  return (
    <ul>
      {users.map((user) => <li key={user.id}>{user.name}</li>)}
    </ul>
  );
}
