00なぜ Props？ — コンポーネントは「引数つきの関数」

Day04・Day05 で作ったコンポーネントは、いつも同じ内容を表示していました。 でも実際のアプリでは「名前だけ違うプロフィールカードを10個」のように、形は同じで中身だけ違うものをたくさん並べたいことがほとんどです。

そこで Props の出番です。コンポーネントは中身が JavaScript の関数でしたよね。 関数が 引数 を受け取って結果を変えるのと同じで、コンポーネントは Props を受け取って表示を変えます。

ふつうの関数

function greet(name) {
  return "こんにちは " + name;
}
greet("花子"); // 引数を渡す

コンポーネント

function Greet(props) {
  return <h1>こんにちは {props.name}</h1>;
}
<Greet name="花子" /> // propsを渡す
同じ発想だと気づくと一気にラクになります。 「引数」が「Props」に、「greet("花子")」が「<Greet name="花子" />」に変わっただけ。 Props は コンポーネントへの唯一の引数だと公式も説明しています。

01Props を渡す（親）と受け取る（子）

Props のやり取りは必ず2か所で起きます。「渡す側（親）」と「受け取る側（子）」です。 まずは一番素朴な書き方から見てみましょう。

① 親：タグの属性のように書いて渡す

function Profile() {
  return (
    <Avatar name="山田 花子" size={100} />
    //      ↑文字列は " "    ↑数値は { }（次のセクションで詳説）
  );
}

HTMLで <img src="..." /> と書くのと同じ要領で、自作の <Avatar> に name と size という名前で値をくっつけて渡しています。この名前は自分で自由に決められます。

② 子：props という1個のオブジェクトで受け取る

function Avatar(props) {
  // 親が渡した値は、props にまとめて入ってくる
  // props = { name: "山田 花子", size: 100 }
  return (
    <div>
      <h2>{props.name}</h2>
      <p>サイズ: {props.size}px</p>
    </div>
  );
}
ポイント: 親がいくつ値を渡しても、子の引数は props という1個のオブジェクトにまとめられて届きます。 だから子の中では props.name、props.size のように 「props.（ドット）＋名前」 で取り出します。 Day05 の演習6で user.name と書いたのと、まったく同じ「ドットで中身を取り出す」操作です。
大文字で始めるのがコンポーネントの約束。 <Avatar> のように先頭が大文字だと「自作コンポーネント」、 <div> のように小文字だと「ふつうのHTMLタグ」とReactは区別します。だから自作コンポーネントは必ず大文字始まりにします。

02分割代入でスッキリ受け取る

毎回 props.name、props.size と「props.」を付けるのは少し面倒です。 そこで実際の現場では、Day02 で学んだ分割代入（ぶんかつだいにゅう）を使って受け取ります。

// ▼ さっきの書き方
function Avatar(props) {
  return <h2>{props.name}</h2>;
}

// ▼ 分割代入バージョン（こちらが定番）
function Avatar({ name, size }) {
  // 引数のところで { } を使って、props の中身を直接取り出す
  return <h2>{name}（{size}px）</h2>;
  // もう props. を付けなくていい！
}
何が起きている？ function Avatar({ name, size }) は、 「届いた props オブジェクトから、name と size という鍵（プロパティ名）の値を取り出して、同じ名前の変数に入れる」という意味です。
Day02 でやった const { name, size } = props; を、引数の場所でいっぺんにやっているだけ。中身はまったく同じです。
どちらの書き方でも動きます。まずは props で受けて props.name と書くのが分かりやすいなら、それでOK。 慣れてきたら { name, size } の分割代入に乗り換えましょう。公式のサンプルもほぼこの形です。

03いろんな値を渡す — 文字列は " "、それ以外は { }

Props にはあらゆる JavaScript の値を渡せます。文字列・数値・真偽値・配列・オブジェクト・関数まで。 渡し方のルールは Day05 で学んだ JSX の属性と同じ——「ただの文字列だけが " "、それ以外はすべて { }」です。

渡したい値書き方ひとこと
文字列name="花子"これだけクォートでOK
数値size={100}{ } で囲む（"100" は文字列になる）
真偽値isAdmin={true}{ } で囲む
配列tags={["A","B"]}{ } の中に配列
オブジェクトperson={{ name:"花子" }}{{ }} になる（理由は下）
変数url={siteUrl}クォートを付けない
function Profile() {
  const person = { name: "山田 花子", job: "エンジニア" };
  return (
    <Card
      title="プロフィール"   // 文字列 → " "
      level={3}              // 数値 → { }
      person={person}        // 変数 → { }
      tags={["React", "CSS"]}  // 配列 → { }
    />
  );
}
オブジェクトの {{ }} は「2重括弧」じゃない。 Day05 の style={{ }} と同じ理屈です。外側の { } は「ここからJS」の合図、内側の { } は「オブジェクトそのもの」。 合わせて person={{ name:"花子" }} になります。
数値を " " で囲むと文字列になります。 size="100" は文字の "100"、 size={100} は数の 100。計算に使うなら必ず { } で渡しましょう。

04デフォルト値 — 渡されなかったときの保険

「size を渡し忘れたら表示が崩れる」——そんな事故を防ぐため、 渡されなかったときに使う初期値（デフォルト値）を子側で決めておけます。分割代入の = を使います。

function Avatar({ name, size = 100 }) {
  // size が渡されなかったら、自動的に 100 が使われる
  return <img width={size} height={size} alt={name} />;
}

<Avatar name="花子" />          // size 省略 → 100 が使われる
<Avatar name="花子" size={200} />  // 200 が使われる
デフォルト値が効くのは「渡されていない（undefined）」ときだけ。 size={0} や size={null} をあえて渡した場合は、デフォルトの 100 ではなく、その 0 や null がそのまま使われます。

05children — タグで「囲んだ中身」を渡す

これまでは name=... のように名前を付けて値を渡しました。 でも <div>...</div> のように、タグで何かを「囲む」形でも値を渡せます。 このとき、囲まれた中身は children（チルドレン＝子ども）という特別な名前の Props で届きます。

function Card({ children }) {
  // タグで囲まれた中身が children に入ってくる
  return <div className="card">{children}</div>;
}

function App() {
  return (
    <Card>                      // ← 開きタグ
      <h2>こんにちは</h2>        // ┐
      <p>これが children です</p>  // ┘この2つが children
    </Card>                     // ← 閉じタグ
  );
}
何がうれしい？ Card は「枠（見た目）」だけを用意して、 中身は使う人が自由に差し込めるようになります。パネル・囲み枠・レイアウトなど、 「外側はそのまま、中身だけ入れ替えたい」場面で大活躍する超重要パターンです。

06Props は読み取り専用（書き換え禁止）

最後に大事なルールを1つ。子コンポーネントは、受け取った Props を自分で書き換えてはいけません。 Props は親から渡された「読み取り専用のメモ」だと考えてください。

function Avatar({ size }) {
  size = 200; // ✕ NG: props を書き換えてはいけない
  return <img width={size} />;
}
なぜ？ Props は「ある瞬間に親から渡された値のスナップショット（写真）」です。 表示を変えたいときは、子が勝手に書き換えるのではなく、親が新しい値を渡し直すのがReactの流儀。 「ユーザー操作に応じて値を変えたい」場合は、次のステップで学ぶ state（ステート） を親に持たせます（Day08 で登場！）。
いまは「Props は上（親）から下（子）へ一方通行で流れる」「子は受け取るだけ」とだけ覚えておけば十分です。 この一方通行のおかげで、データの流れが追いやすく、バグが減ります。

07＋α スプレッドで props をまとめて転送

（余裕があれば、の発展トピックです。難しければ読み飛ばしてOK）

受け取った props をまるごと別のコンポーネントに渡したいとき、Day02 で学んだスプレッド構文 ... が使えます。

function Profile(props) {
  return (
    <div className="frame">
      <Avatar {...props} />  // props の中身を全部 Avatar に渡す
    </div>
  );
}
便利だけど使いすぎ注意。 {...props} を多用すると「結局どの値が渡っているのか」が分かりにくくなります。 公式も「多用は設計を見直すサイン」と注意しています。基本は1つずつ名前を付けて渡しましょう。
