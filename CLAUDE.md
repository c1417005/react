# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## このリポジトリについて

React を体系的に学習するための学習用リポジトリです。コードベースではなく、学習の進行そのものが目的です。

## 対象と前提

- **対象言語/フレームワーク**: React（まず JavaScript ベースで進め、後で TypeScript に移行する予定）
- **学習者のレベル**:
  - JavaScript: 触りだけ
  - CSS: 基礎を少し
  - TypeScript: 少し
- 前提知識が浅いことを踏まえ、専門用語には初出時に簡単な補足を添える。

### 解説の詳しさ（重要）

- **初学者向けに、解説はできるだけ細かく丁寧に書く（Day 02 以降に適用）。**
- 基礎構文も「知っている前提」にしない。特に以下のような土台部分は、初めて出てきたときに一度きちんと説明する:
  - `for` 文、`for...of`、`for...in` の違いと使い分け
  - 配列・オブジェクトの基本操作
  - 関数の定義と呼び出し、引数・戻り値
- コードは1行ずつ「何をしているか」をコメントや文章で補い、結果がどうなるかも示す。
- 「なぜそう書くのか」「他の書き方との違い」まで触れて、丸暗記ではなく理解につなげる。

## 学習スタイル（ディレクトリ構成）

- 各 Day ごとにディレクトリを作る（例: `Day01/`, `Day02/` ...）。
- 各 Day のディレクトリには、教材を **2種類のファイルに分けて** 配置する:
  - **解説・例題 → HTML ファイル**（例: `Day02/lesson.html`）
  - **演習課題 → 実行可能なコードファイル**（例: `Day02/exercise.mjs`。`node` などでそのまま実行できる状態）

### 解説・例題（HTML）の書き方

- **Day 02 以降は、解説と例題（説明用のサンプルコード）を HTML ファイルで書く。**
  （Day 01 は `.mjs` のコメント形式で作成済み。さかのぼって作り直す必要はない。）
- ページ先頭に **目次（TOC）** を置き、各セクションへアンカーリンクで飛べるようにする。
- 各セクションの末尾に **「目次に戻る」リンク** を置き、目次へ戻れるようにする。
- 例題のコードは `<pre><code>` などで読みやすく表示する。
- ブラウザでそのまま開ける単一HTML（外部ビルド不要）を基本とする。

## 演習課題のルール

- 演習課題は **「// ここにコードを書いてください」** の形で空欄を残す。
- **回答（模範解答）は出さない。**
- 学習者が書いたコードを **レビューする方式** で進める。
  - 良い点・改善点・なぜそうなるかを説明する。
  - すぐに正解コードを書き与えるのではなく、気づきを促す。

## 進捗管理

- 進捗は `PROGRESS.md` で管理する。
- **Day を完了するごとに `PROGRESS.md` を更新する。**

## 一次情報とハルシネーション防止

- **公式ドキュメント（https://react.dev/learn）を一次情報として参照する。**
- 記憶だけに頼らず、不確かな点は公式ドキュメントを確認してから答える（ハルシネーションを避ける）。
- 学習者が不明点を質問したときは、**まず公式ドキュメントの該当箇所を示してから** 解説する。
