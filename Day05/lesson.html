0. なぜ JSX？ — 見た目とロジックを同じ場所に

昔の Web 開発では「HTML（見た目）」と「JavaScript（動き）」を別ファイルに分けるのが普通でした。 しかし実際には、ボタンの見た目とそのクリック処理のように、両者は密接に関わっています。 React は「関係するものは一緒に置く」という考えで、見た目（マークアップ）とロジックをコンポーネントという1つの単位にまとめます。その「見た目」を書くための記法が JSX です。

JSX は HTML にそっくりですが、中身は JavaScript です。だから { } を使えば、変数・計算・条件などをそのまま見た目に差し込めます（Day04 で学んだ通り）。 「HTML に JS の力が加わったもの」とイメージすると分かりやすいです。

1. JSX の3大ルール

JSX は HTML に似ていますが、JavaScript なので守るべきルールが3つあります。Day04 でも触れましたが、ここで理由までしっかり押さえましょう。

ルール① 必ず「1つの親要素」で包む

コンポーネントが return できる JSX は1個だけです。要素を並べたいときは <div> やフラグメント <></> で包みます。

// ✕ NG: 兄弟要素を2つ返している
return (
  <h1>タイトル</h1>
  <p>本文</p>
);

// ◯ OK: 1つの親で包む
return (
  <div>
    <h1>タイトル</h1>
    <p>本文</p>
  </div>
);
なぜ？ JSX は内部的に「1個のオブジェクト」に変換されます。関数が値を2個同時に return できないのと同じで、JSX も「1個」にまとめる必要があるのです。

ルール② すべてのタグを閉じる

HTML では <br> や <img> を閉じなくても動きますが、JSX では必ず閉じます。 中身のないタグは <img /> のように末尾に /（自己閉じ）を付けます。

// ✕ <img src="cat.png">        ← 閉じていない
// ◯ <img src="cat.png" />     ← 自己閉じ
// ◯ <li>りんご</li>            ← 開いて閉じる

ルール③ 属性は camelCase（キャメルケース）で書く

JSX の属性名の多くは、HTML と少し違って「途中で大文字にする」書き方（camelCase）になります。 特に class は JavaScript の予約語なので className と書きます（詳しくは セクション7）。

// HTML:  <label class="title" for="name" tabindex="0">
// JSX :  <label className="title" htmlFor="name" tabIndex={0}>
camelCase（キャメルケース）とは: fontSize のように「単語の区切りを大文字にする」書き方です。 ラクダ（camel）の背中のコブのように見えることから命名されています。onclick → onClick なども同じ仲間です。

2. フラグメント <></> — 余計な箱を増やさず包む

ルール①で「1つの親で包む」と学びましたが、そのために毎回 <div> を足すと、 画面に意味のない箱が増えてしまいます。そんなときはフラグメントを使います。

function Info() {
  return (
    <>
      <h2>お知らせ</h2>
      <p>本日は晴天です</p>
    </>
  );
}
<>...</> は「中身をまとめるだけで、自分自身は画面に何も出さない」特別な入れ物です。 <div> で包むと HTML に <div> が増えますが、フラグメントなら増えません。「1個に包む条件は満たしたいが、余計なタグは出したくない」ときの定番です。

3. 属性に「文字列」を渡す（" "）

属性の値がただの文字列なら、HTML と同じくダブルクォート " " で書きます。

  return <img className="avatar" src="cat.png" alt="うちの猫" />;
}

4. 属性に「JavaScript」を渡す（{ }）

属性の値を変数や式で決めたいときは、" " の代わりに { } で囲みます。 Day04 で学んだ「JSX の中身に値を差し込む { }」と同じものを、属性にも使えると考えてください。

function Avatar() {
  const imageUrl = "https://example.com/cat.png";
  const altText = "うちの猫";
  return <img src={imageUrl} alt={altText} />;
  //          ↑ 変数        ↑ 変数
}
" " と { } を混ぜない。 src="{imageUrl}" と書くと、「{imageUrl}」という文字列そのものが src になってしまいます（変数として扱われない）。 変数を渡すならクォートは付けず src={imageUrl} です。

文字列と変数を混ぜたいとき

「https://.../ + ユーザーID」のようにつなげたいときは、{ } の中でテンプレートリテラル（バッククォート）を使います。

const userId = 42;
return <img src={`https://example.com/users/${userId}.png`} />;
//            ↑ { } の中は JavaScript なので、バッククォートが使える
Day04 の演習5で「JSX のテキストにバッククォートを書いても効かない」と学びましたね。 ポイントは「{ } の中＝JavaScript」。{ } の中ならテンプレートリテラルが正しく働きます。場所の違いが鍵です。

5. style は二重波括弧 {{ }}

インライン（その場で書く）スタイルを当てるときは、少し特殊な見た目になります。

function Title() {
  return (
    <h1 style={{ color: "red", fontSize: "24px" }}>
      赤い見出し
    </h1>
  );
}
なぜ波括弧が2つ？ 分解すると理解できます。
  • 外側の { } … 「ここから JavaScript ですよ」の合図（属性に JS を渡す { }）。
  • 内側の { } … JavaScript のオブジェクト（{ キー: 値 } の形）。
つまり style={ <オブジェクト> } で、その中身が { color: "red" } というオブジェクト。 合わせて style={{ color: "red" }} になります。「2つ」ではなく「JS の合図」＋「オブジェクト」です。
CSS プロパティ名も camelCase。 CSS の font-size はハイフンが使えないので fontSize、 background-color は backgroundColor と書きます。値は文字列（"24px"）にします。

6. { } に入れられるもの・入れられないもの

{ } の中に書けるのは「式（evaluate すると1つの値になるもの）」だけ、というのが最重要ルールです（Day04 でも登場）。

入れられる（式）例
変数{name}
計算{price * 1.1}
関数呼び出し{formatDate(today)}
三項演算子{ok ? "○" : "×"}
&& による条件表示{count > 0 && <p>あり</p>}
配列の map{items.map(...)}
入れられない（文）理由
if (...) { }「文」であって値にならない → 三項演算子か && を使う
for (...) { }「文」であって値にならない → map を使う
const x = 1代入は「文」 → JSX の外（return の前）で書く

&& による条件表示（片方だけ出す）

「条件が true のときだけ表示、false なら何も出さない」場合は && が便利です。

function Inbox() {
  const unread = 3;
  return (
    <div>
      <h2>受信箱</h2>
      {unread > 0 && <p>未読が {unread} 件あります</p>}
    </div>
  );
}
// unread が 3 なので「未読が 3 件あります」が表示される
// もし unread が 0 なら、&& の左が false になり、<p> は表示されない
&& の落とし穴: 左側が数値の 0 だと、0 がそのまま画面に表示されてしまいます。 「個数が 0 のとき何も出したくない」なら {count > 0 && ...} のように必ず true/false になる条件にしましょう（{count && ...} は危険）。

7. HTML と名前が違う属性（className など）

JSX は JavaScript なので、JavaScript の予約語とぶつかる属性名は別名になっています。よく使うものだけ覚えれば十分です。

HTMLJSXメモ
classclassNameclass は予約語。最頻出
forhtmlForfor も予約語（ループの for）
onclickonClickイベントは camelCase（Day08 で本格的に）
tabindextabIndex多くの属性が camelCase に
全部を暗記する必要はありません。class → className だけは即答できるようにしておけば、残りは出てきたときに調べればOKです。 React は class と書くと開発中に警告を出してくれます。

8. JSX の中のコメント {/* */}

JSX の中（タグの中身）にコメントを書くときは、{/* ... */} の形にします。 これは「{ } の中に JavaScript のコメントを入れている」と考えると納得できます。

return (
  <div>
    {/* これは JSX 内のコメント。画面には出ない */}
    <h1>こんにちは</h1>
  </div>
);
タグの中身で // コメント や <!-- コメント --> はそのまま文字として表示されたりエラーになります。 JSX の中では必ず {/* */} を使ってください。
